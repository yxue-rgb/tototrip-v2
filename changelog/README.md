# 📦 TotoTrip 版本归档

> 每个版本的开发日志、截图、变更记录。用于证明开发流程、投资人展示、知识产权保护。

## 文件夹结构

```
changelog/
├── README.md              ← 本文件
├── v2.0/
│   ├── CHANGELOG.md       ← 版本变更日志
│   ├── screenshots/       ← 产品截图
│   │   ├── homepage.png
│   │   ├── chat.png
│   │   ├── map.png
│   │   └── mobile.png
│   └── deploy-log.md      ← 部署记录
├── v2.1/
│   ├── CHANGELOG.md
│   ├── screenshots/
│   └── deploy-log.md
└── ...
```

## 归档规范

### 每个版本必须包含：
1. **CHANGELOG.md** — 变更内容（新增功能、修复 bug、改进项）
2. **screenshots/** — 至少 4 张截图（首页、核心功能页、移动端、对比前后）
3. **deploy-log.md** — 部署时间、部署命令、Zeabur deployment ID

### 截图命名规范：
- `homepage-desktop.png` — 首页桌面端
- `homepage-mobile.png` — 首页移动端
- `chat-desktop.png` — 聊天页桌面端
- `chat-mobile.png` — 聊天页移动端
- `map-view.png` — 地图视图
- `feature-xxx.png` — 具体功能截图

### 版本号规则：
- v2.0 — 基础版（当前）
- v2.1 — AI 增强版
- v2.2 — 社交版
- v2.3 — 商业化版
- v3.0 — 全栈版
