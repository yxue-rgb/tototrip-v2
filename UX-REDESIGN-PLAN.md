## 一、竞品分析：MindTrip 做对了什么

### 核心 UX 理念
MindTrip 的设计哲学 = **对话即旅行规划，地图是第一等公民**

### MindTrip 关键设计模式

| 模式 | MindTrip 做法 | 为什么好 |
|------|-------------|---------|
| **双栏布局** | 左侧对话 + 右侧实时地图 | 用户问什么，地图同步标注，空间感知极强 |
| **地点卡片** | 名称 + 评分 + 真实图片 + ❤️收藏 + ➕加入行程 | 信息密度高，操作直觉 |
| **顶栏快捷筛选** | Where / When / 人数 / Budget 按钮 | 一眼知道在规划什么，随时可改 |
| **地图标记** | 分类图标（🍜餐厅/🏛景点/🏨酒店）+ 聚合 | 不同类型一目了然 |
| **对话流** | AI 回复中嵌入可交互的地点卡片 | 不是纯文字，是可操作的结构化内容 |
| **侧边栏导航** | Chats / Trips / Explore / Saved / Inspiration | 清晰的功能分区 |
| **Create a trip** | 一键把对话结果转为行程 | 从聊天到计划无缝衔接 |

## 二、TotoTrip 当前 UX 问题诊断

### 🔴 严重问题

1. **地图是附属品，不是核心**
   - 地图只在 AI 回复推荐地点后才显示一个小框
   - 没有常驻地图面板
   - 用户无法在对话同时感知空间关系

2. **聊天页面太封闭**
   - 全屏深色背景对话界面，像在聊天软件而非旅行工具
   - 没有侧边栏、没有导航、没有上下文
   - 用户无法同时看到地图和对话

3. **地点推荐是纯文本**
   - AI 返回的地点没有图片、没有评分可视化
   - LocationCard 有但体验粗糙
   - 缺少收藏/加入行程的快捷操作

4. **首页和聊天断裂**
   - 首页很华丽但信息密度低
   - 点进聊天后完全不同的界面风格
   - 没有 MindTrip 那种 "For you in [城市]" 的个性化推荐

### 🟡 体验问题

5. **没有行程管理视图** — 聊天了半天，结果呢？没有 Trip 概念
6. **没有 Explore 发现页** — 用户不知道可以做什么
7. **移动端对话+地图无法同时呈现** — 需要 tab 切换策略
8. **输入框功能单一** — 没有语音、附件、预设 prompt

## 三、UX 重建计划

### Phase 1：核心框架重构（最高优先级）⚡

**目标**：建立 MindTrip 级别的双栏聊天+地图布局

#### 1.1 双栏布局系统
```
┌──────────────────────────────────────────────────┐
│ [Logo] [Destinations] [Trips] [Explore]    [Login]│  ← 顶部导航
├────────────────────────┬─────────────────────────┤
│                        │                         │
│   💬 Chat Panel        │    🗺️ Map Panel         │
│                        │                         │
│  [AI对话流]             │  [Leaflet 实时地图]       │
│  [地点卡片嵌入]          │  [分类标记]              │
│  [收藏/加行程操作]       │  [点击标记→高亮卡片]      │
│                        │                         │
│                        │                         │
├────────────────────────┴─────────────────────────┤
│  [输入框] [📎] [🎤]                     [发送]     │
└──────────────────────────────────────────────────┘
```

- **桌面端**：50/50 或 60/40 左右分栏
- **移动端**：默认对话全屏 + 底部 "Map" tab 可切换
- 地图和对话**联动** — AI 推荐地点时地图自动标记+飞过去

#### 1.2 顶栏快捷筛选条
```
[📍 Beijing] [📅 3 days] [👤 2] [💰 Budget]  [Create Trip ➜]
```
- 在聊天页顶部显示当前规划参数
- 可以随时点击修改

#### 1.3 增强版 LocationCard
```
┌──────────────────────────────────┐
│ [📸 真实图片轮播]                  │
│                                  │
│ 🏛 Forbidden City         ⭐ 4.7 │
│ Attraction · UNESCO World Site   │
│ 📍 Dongcheng District           │
│ ⏱ 3-4 hours · 💰 ¥60            │
│                                  │
│ [❤️ Save]  [➕ Add to Trip]      │
└──────────────────────────────────┘
```

### Phase 2：地图体验升级 🗺️

#### 2.1 常驻交互地图
- 地图始终显示在右侧面板
- AI 每推荐一个地点 → 地图实时添加标记（动画飞入）
- 点击地图标记 → 左侧对话自动滚动到对应卡片（高亮）
- 点击对话卡片 → 地图 flyTo 对应位置

#### 2.2 分类标记系统
- 🍜 餐厅 = 橙色
- 🏛 景点 = 蓝色
- 🏨 酒店 = 紫色
- 🚇 交通 = 绿色
- 🛍 购物 = 粉色
- 聚合标记（zoom out 时合并）

#### 2.3 地图图层切换
- CARTO Voyager（默认，英文清爽）
- 卫星视图（看地形）

---

### Phase 3：信息架构重组 📐

#### 3.1 导航系统
```
[toto 🐕] [Chat] [Trips] [Explore] [Saved]    [Login]
```

#### 3.2 Explore 发现页
- "For you in [目的地]" 推荐流
- 分类浏览：景点 / 美食 / 文化 / 交通攻略
- 热门行程模板

#### 3.3 Trips 行程管理
- 从对话中一键创建行程
- 按天拖拽排序
- 每天时间线视图
- 导出 PDF

#### 3.4 Saved 收藏
- 所有收藏的地点
- 按城市/类别分组

---

### Phase 4：对话体验优化 💬

#### 4.1 结构化 AI 回复
- AI 回复不再是纯文本 + 底部地图
- 而是：文字描述 → 嵌入地点卡片 → 地图同步
- 参考 MindTrip：回复中间直接插入可交互的地点卡片

#### 4.2 输入增强
- 语音输入（重要！旅途中打字不方便）
- 图片上传（拍菜单翻译）
- 预设 prompts / 快捷操作

#### 4.3 对话上下文
- 显示当前在规划哪个城市/行程
- 快捷切换到其他对话

## 四、视觉设计方向

### 从当前深色主题 → 明亮专业主题

| 维度 | 当前 | 目标 |
|------|------|------|
| 主色调 | 深绿/暗色 | 白色 + 品牌色点缀 |
| 背景 | 深色渐变 | 白/浅灰，干净清爽 |
| 排版 | 还行 | 更大的标题、更好的层次 |
| 图片 | 靠 AI 文字描述 | 真实照片、高质量图库 |
| 地图 | 小窗口附属 | 右半屏常驻 |
| 整体感觉 | 聊天 app | 专业旅行规划工具 |

### 保留的品牌元素
- toto 🐕 吉祥物（但从"满屏中心"改为左上角小 Logo）
- 品牌色（可微调）
- "Woof!" 的友好语气
