# 🌍 途图旅行 TotoTrip — Sprint Brief v1.0

> **目标**: 从 demo 级别升级到投资人 pitch + 真实用户可用的产品
> **时间**: 持续迭代，不要停
> **标准**: 对标 MindTrip / Layla.ai 的产品质量

---

## 🎯 两大核心痛点（Master 定义）

### 痛点 1: UI/UX 太基础，没有吸引力
- 当前首页就是一个基础聊天界面，毫无品牌感
- 地图功能几乎不可用
- 没有 inspiration（灵感）板块
- 没有视觉层次和设计语言
- 缺少动画、micro-interaction

### 痛点 2: AI 交互不够专业
- System prompt 太简单（"Be helpful!"）
- 缺少品牌人格和专业度
- 没有结构化的旅行规划输出（行程表、预算表）
- 没有引导性对话流（了解需求 → 推荐 → 细化 → 确认）
- 缺少中国旅行特有知识（支付、VPN、交通、文化）

---

## 📋 Sprint 任务清单（按优先级）

### Phase 1: 视觉重塑 + 品牌升级 ⭐⭐⭐
1. **全新首页设计**
   - Hero section：大图/视频背景 + 搜索框 + 标语
   - "Where do you want to go in China?" 引导
   - 热门城市卡片（北京/上海/成都/西安/桂林等）
   - 用户评价/数据展示
   - 品牌故事 section

2. **设计系统建立**
   - 色彩方案（参考: 中国红 + 现代蓝灰 + 金色点缀）
   - Typography（中英文双语 font stack）
   - 组件库（Button, Card, Badge, Modal 统一风格）
   - 间距、圆角、阴影标准化
   - 动画 motion design（Framer Motion）

3. **品牌标识**
   - Logo 设计或占位
   - favicon + OG 图片
   - 品牌色应用到全站

### Phase 2: AI 交互重塑 ⭐⭐⭐
1. **System Prompt 重写**（最关键！）
   ```
   你是途图旅行的 AI 旅行顾问，专为来中国旅行的外国人服务。
   
   人格特质：
   - 热情但专业，像一个住在中国多年的当地朋友
   - 了解中西方文化差异，能预判外国人的困惑
   - 实用主义 — 不只说"好吃"，要说怎么点餐、怎么付款
   
   交互原则：
   - 第一次对话：了解旅行目的、时间、预算、兴趣
   - 推荐时：给结构化卡片（名称、评分、价格、距离、一句话点评）
   - 主动提醒中国特有注意事项（现金vs微信、VPN、交通卡等）
   - 输出行程时用 day-by-day 格式，包含时间轴
   
   中国旅行专业知识：
   - 支付系统（微信/支付宝 外国人如何开通）
   - 交通（高铁购票、地铁、打车 Didi）
   - 通讯（VPN、eSIM、Wi-Fi）
   - 住宿（外国人可住的酒店要求）
   - 餐饮（点餐方式、饮食禁忌、推荐体验）
   - 文化礼仪（送礼、谈价、社交规范）
   ```

2. **引导式对话流**
   - Onboarding：3-5 个问题了解用户（去哪？多久？几个人？预算？兴趣？）
   - Quick actions（快捷按钮）："Plan my trip" / "Find restaurants" / "Help with transportation"
   - 结构化输出：LocationCard 升级、DayPlan 组件、BudgetTable 组件

3. **AI 后端切换**
   - Anthropic → Gemini 2.5 Pro（统一技术栈，更强中文能力）
   - 优化 prompt 的 JSON 输出格式
   - 添加重试 + fallback 策略

### Phase 3: 地图重做 ⭐⭐
1. **Leaflet/Mapbox 升级**
   - 自定义地图样式（不是默认 OpenStreetMap 瓦片）
   - 美观的自定义标记（分类图标：餐厅🍜/景点🏛️/酒店🏨/交通🚄）
   - 路线连线 + 动画
   - 点击标记显示详情卡片（overlay）
   - 地图 ↔ 列表视图切换

2. **行程 Timeline 视图**
   - Day-by-day 可视化
   - 拖拽调整顺序
   - 时间估算 + 交通方式
   - 导出/分享

### Phase 4: 功能补全 ⭐
1. **Inspiration 灵感页面**
   - 精选行程模板（7天经典北京上海、14天深度西南等）
   - 照片墙 + 故事
   - 一键 "Use this plan" → 导入到自己的行程

2. **用户体验**
   - 深色模式
   - 多语言（英/中/日/韩）
   - PWA 支持
   - 响应式完善

3. **数据持久化**
   - 确认 Supabase 连接正常
   - 聊天历史保存
   - 行程保存/导出

## 🔧 技术决策

- **AI**: Gemini 2.5 Pro（替换 Anthropic/DeepSeek/Groq 多 provider 混乱）
- **地图**: 继续用 Leaflet（免费），但换更好的瓦片样式
- **部署**: Vercel（先跑起来，后续考虑域名）
- **端口**: dev 用 3001（避免和 Wine Advisor 的 3000 冲突）

## ⚠️ 注意事项

1. 每完成一个 Phase → 自我测试 → 写体验报告 → 继续下一轮
2. 代码质量：组件化、TypeScript 严格模式、注释
3. 移动端优先（很多用户会手机访问）
4. 性能：图片懒加载、代码分割
5. .env.local 需要更新（加 GEMINI_API_KEY，移除旧的 Anthropic key）
